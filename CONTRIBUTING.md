# Contributing to Hassan Ali — Next.js Portfolio

First off, thank you for considering contributing! 🎉 Even though this is a personal portfolio, community feedback and contributions are genuinely welcome — especially around performance, accessibility, and SEO improvements.

## How Can I Contribute?

### 🐛 Reporting Bugs
- Use the [Bug Report template](./.github/ISSUE_TEMPLATE/bug_report.md)
- Check existing issues first to avoid duplicates
- Include steps to reproduce, expected vs actual behavior, and screenshots if relevant

### ✨ Suggesting Enhancements
- Use the [Feature Request template](./.github/ISSUE_TEMPLATE/feature_request.md)
- Explain the problem your suggestion solves and why it'd be valuable

### 🔧 Pull Requests

1. Fork the repo and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. Follow the existing code style (ESLint + Prettier are configured — run `npm run lint`)
3. Write clear, descriptive commit messages (prefer [Conventional Commits](https://www.conventionalcommits.org/), e.g. `feat: add dark mode toggle`)
4. Make sure the build passes: `npm run build`
5. Push to your fork and open a Pull Request against `main`
6. Describe **what** you changed and **why** in the PR description

### Code Style

- TypeScript strict mode — no `any` unless justified
- Tailwind utility classes over custom CSS where possible
- Keep components small, typed, and single-responsibility
- Accessibility is non-negotiable — semantic HTML, proper ARIA where needed, keyboard navigability

## Code of Conduct

Be respectful, constructive, and welcoming. Disagreements about code are fine; disrespect toward people is not.
